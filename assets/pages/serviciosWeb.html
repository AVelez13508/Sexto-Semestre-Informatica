SERVICIOS WEB

Un servicio web es una interfaz que permite que los dispositivos se comuniquen a través de la Red (Internet o Intranet). Utiliza protocolos estándares y formatos basados en texto (como XML o JSON) para intercambiar datos, lo que garantiza la interoperabilidad.

CONCEPTOS CLAVE

Interoperabilidad:

Es la capacidad de que sistemas heterogéneos se conecten. No importa si usas Windows, Linux o iOS; el servicio web nivela el campo de juego.

Protocolos de comunicación:

Son las "reglas del juego. Los más comunes son:

HTTP/HTTPS:

El vehículo que transporta la información.

SOAP (Simple Object Access Protocol):

Un protocolo robusto basado en XML, muy estricto y seguro.

REST (Representational State Transfer):

Un estilo arquitectónico más flexible, que utiliza HTTP y formatos como JSON o XML.

Formatos de intercambio de datos:

XML (eXtensible Markup Language):

Estructura de etiquetas (parecido al HTML) usado principalmente en SOAP.

JSON (JavaScript Object Notation):

Un formato ligero y fácil de leer, muy popular en REST.

WSDL (Web Services Description Language):

Es el "manual de instrucciones" de un servicio web (usado en SOAP). Le dice al cliente qué funciones están disponibles y cómo llamarlas.
